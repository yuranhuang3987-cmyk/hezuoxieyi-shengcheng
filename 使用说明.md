# 软件著作权合作协议生成器 - Web 版

**版本：** 1.0.0
**创建时间：** 2026-02-28

---

## 📋 功能介绍

将原有的 Python 桌面程序改造为 Web 应用，支持：

- ✅ 上传软件著作权申请表（.docx）
- ✅ 自动提取软件信息并预览
- ✅ 一键生成合作协议
- ✅ 在线下载生成的协议
- ✅ 保存历史记录，随时查看

---

## 🚀 快速开始

### 系统要求

- **Python 3.7+** （后端运行环境）
- **Node.js 16+** （前端运行环境）

### 安装步骤

#### 方式一：自动安装（推荐）

1. **启动后端**
   - 双击 `启动后端.bat`
   - 等待依赖安装完成
   - 看到提示 "服务启动: http://localhost:5000" 表示成功

2. **启动前端**（新开一个终端）
   - 双击 `启动前端.bat`
   - 等待依赖安装完成
   - 浏览器自动打开 http://localhost:3000

3. **开始使用**
   - 上传 .docx 文件
   - 预览提取的信息
   - 点击"生成协议"
   - 下载生成的协议

#### 方式二：手动安装

**后端：**
```bash
cd backend
pip install -r requirements.txt
python app.py
```

**前端：**
```bash
cd frontend
npm install
npm start
```

## 📖 使用说明

### 第一步：上传申请表

1. 在首页点击或拖拽上传 `.docx` 格式的软件著作权申请表
2. 系统自动解析并提取信息

### 第二步：确认信息

查看提取的软件信息：
- 软件名称
- 版本号
- 开发完成日期
- 协议签署日期（自动计算，提前3个月）
- 著作权人信息（区分个人/单位）

### 第三步：生成协议

1. 确认信息无误后，点击"生成协议"
2. 系统根据著作权人数量自动选择对应模板（2-13方）
3. 生成完成后，点击"下载合作协议"

### 查看历史记录

1. 点击顶部菜单"历史记录"
2. 查看所有生成的协议
3. 可以下载、查看详情或删除记录

## 🔧 API 接口

### 预览接口
```
POST /api/preview
参数: file (multipart/form-data)
返回: 提取的软件信息
```

### 生成接口
```
POST /api/generate
参数: JSON (preview数据 + upload_path)
返回: 下载链接
```

### 下载接口
```
GET /api/download/<filename>
返回: 协议文件
```

### 历史记录接口
```
GET /api/history?page=1&per_page=10
返回: 历史记录列表
```

## ⚙️ 配置说明

### 后端配置

在 `backend/app.py` 中可以修改：

```python
# 最大上传文件大小（默认 16MB）
app.config["MAX_CONTENT_LENGTH"] = 16 * 1024 * 1024

# 服务端口（默认 5000）
app.run(host="0.0.0.0", port=5000, debug=True)
```

### 前端配置

在 `frontend/src/pages/Home.js` 中修改后端地址：

```javascript
const response = await axios.post('http://localhost:5000/api/preview', formData);
```

## 🐛 常见问题

### 1. 后端启动失败

**原因：** 未安装 Python 或版本过低

**解决：** 安装 Python 3.7 或更高版本

### 2. 前端启动失败

**原因：** 未安装 Node.js 或版本过低

**解决：** 安装 Node.js 16 或更高版本

### 3. 上传文件后无响应

**原因：** 后端未启动

**解决：** 先启动后端（`启动后端.bat`），再启动前端

### 4. 提取信息失败

**原因：** 文件格式不正确

**解决：** 确保上传的是 `.docx` 格式的软件著作权申请表

### 5. 生成的协议无法下载

**原因：** 浏览器拦截弹窗

**解决：** 允许浏览器弹窗，或右键"下载"选择"另存为"

1. **生产环境部署时：**
   - 关闭 Flask debug 模式
   - 使用 HTTPS
   - 添加用户认证
   - 限制上传文件大小和类型

2. **数据安全：**
   - 定期备份数据库
   - 设置文件访问权限
   - 定期清理过期文件
