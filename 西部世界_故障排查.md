# 西部世界 VPN - WSL2 连接故障排查

- ✅ Windows 防火墙规则已添加
- ❌ WSL2 仍无法连接到代理端口

## 可能的问题

### 1. 西部世界需要重启
**操作**：
- 完全关闭西部世界客户端
- 重新启动

### 2. 端口号不正确
**如何确认真实端口**：

#### 方法 A：在西部世界中查看
1. 打开西部世界
2. 设置 → 入站设置
3. 查看实际监听的端口号

#### 方法 B：在 Windows 中查看
在 **PowerShell** 中执行：
```powershell
netstat -ano | findstr "LISTENING" | findstr "0.0.0.0"
```
找到西部世界相关的端口

#### 方法 C：任务管理器
1. 打开任务管理器
2. 找到西部世界进程
3. 右键 → 打开文件所在位置
4. 确认是哪个进程在监听

### 3. 需要添加系统规则

你提到的"系统规则"可能需要添加 WSL2 网段的规则：

在西部世界的系统规则中添加：
```
! 允许 WSL2 网段
@@172.29.160.0/20
@@172.16.0.0/12
@@10.0.0.0/8
@@192.168.0.0/16
```

### 4. Windows 防火墙出站规则

**在 PowerShell（管理员）中执行**：
```powershell
# 允许出站连接
New-NetFirewallRule -DisplayName "西部世界出站" -Direction Outbound -LocalPort 21882 -Protocol TCP -Action Allow
```

### 5. 临时测试：关闭防火墙

**仅用于测试**（记得重新开启）：
```powershell
Set-NetFirewallProfile -Profile Public,Private -Enabled False
```

然后在 WSL2 中测试：
```bash
curl -x http://172.29.160.1:21882 -I https://www.google.com
```

如果成功，说明是防火墙问题。
