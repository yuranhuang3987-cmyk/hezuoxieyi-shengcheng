## 📋 执行摘要

通过对多个渠道的系统性调研，发现 OpenClaw 安装过程中的问题主要集中在 **环境依赖**、**权限配置**、**版本兼容性** 三个核心领域。

**最常见问题 TOP 5：**
1. Node.js 版本不符合要求（需要 22+，但用户使用了 18 或 20）
2. npm 全局安装权限错误（EACCES）
3. PATH 配置错误导致 `openclaw: command not found`
4. 端口冲突（18789 端口被占用）
5. 升级后配置不兼容

**关键发现：**
- 2026年2月存在严重安全漏洞（CVE-2026-25253，CVSS 8.8）
- ClawHub 上发现 341 个恶意 Skills
- 从 clawdbot 升级到 openclaw 的用户经常遇到遗留进程冲突

## 📊 问题分类与解决方案

### 1️⃣ 环境依赖问题

#### 问题 1.1：Node.js 版本不符合要求

**错误信息：**
```
OpenClaw requires Node.js 22 or newer. You have v18.x.x
```

**问题原因：**
- OpenClaw 使用了 Node 22+ 的特性
- 旧版本（18/20）会导致模块解析错误

**解决方案：**

**macOS/Linux (nvm)：**
```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash
source ~/.bashrc  # 或 ~/.zshrc
nvm install 22
nvm use 22
nvm alias default 22
```

**Windows：**
- 从 nodejs.org 下载 Node.js 22+ 安装包
- 或使用 WSL2（推荐）

**验证：**
```bash
node --version  # 应显示 v22.x.x 或更高
```

**检查位置：** 查看当前 Node 版本 → 确认是否 ≥ 22

---

#### 问题 1.2：构建工具缺失（Linux）

**错误信息：**
```
make: command not found
```

**问题原因：**
- npm 依赖需要编译原生模块
- 缺少 gcc、make、python3 等构建工具

**解决方案：**

**Ubuntu/Debian：**
```bash
sudo apt-get install build-essential git python3
```

**Fedora/RHEL：**
```bash
sudo dnf groupinstall 'Development Tools'
```

**Arch Linux：**
```bash
sudo pacman -S base-devel
```

**检查位置：** 尝试安装 → 如果失败，检查构建工具

---

#### 问题 1.3：Xcode Command Line Tools 缺失（macOS）

**错误信息：**
```
xcode-select: error: tool requires Xcode
```

**问题原因：**
- macOS 编译原生依赖需要 Xcode 工具
- sharp 等库需要编译

**解决方案：**
```bash
xcode-select --install
```

**Apple Silicon (M1/M2/M3) 额外步骤：**
```bash
softwareupdate --install-rosetta
```

**检查位置：** macOS 安装失败 → 检查 Xcode 工具

### 2️⃣ 权限问题

#### 问题 2.1：npm 全局安装权限错误

**错误信息：**
```
npm ERR! code EACCES
npm ERR! syscall mkdir
npm ERR! path /usr/local/lib/node_modules/openclaw
npm ERR! errno -13
npm ERR! Error: EACCES: permission denied
```

**问题原因：**
- npm 默认全局目录需要 root 权限
- 使用 sudo 会导致文件归属权问题

**解决方案（推荐顺序）：**

**方案 1：使用 nvm（最佳实践）**
```bash
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
nvm install 22
npm install -g openclaw@latest
```

**方案 2：修复 npm 权限**
```bash
mkdir ~/.npm-global
npm config set prefix '~/.npm-global'
echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
npm install -g openclaw@latest
```

**方案 3：临时使用 sudo（不推荐）**
```bash
sudo npm install -g openclaw@latest
# 然后修复归属权
sudo chown -R $USER:$(id -gn $USER) ~/.npm
```

**Windows：**
- 以管理员身份运行 PowerShell

**检查位置：** npm install 报错 → 检查权限

---

#### 问题 2.2：Docker 容器权限问题

**错误信息：**
```
Error: EACCES: permission denied
```

**问题原因：**
- 容器用户（UID 1000）无法访问挂载卷

**解决方案：**
```bash
# 修复宿主机目录归属权
sudo chown -R 1000:1000 ~/.openclaw ~/openclaw/workspace

# 重启容器
cd ~/openclaw
docker compose restart
```

**检查位置：** Docker 部署 → 检查卷挂载权限

### 3️⃣ 配置问题

#### 问题 3.1：PATH 配置错误

**错误信息：**
```
openclaw: command not found
```

**问题原因：**
- npm 全局 bin 目录不在 PATH 中
- 安装成功但系统找不到命令

**解决方案：**

**查找 npm 全局路径：**
```bash
npm config get prefix
# 或
npm prefix -g
```

**添加到 PATH：**

**Bash：**
```bash
echo 'export PATH="$(npm config get prefix)/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

**Zsh：**
```bash
echo 'export PATH="$(npm config get prefix)/bin:$PATH"' >> ~/.zshrc
source ~/.zshrc
```

**Windows：**
- 添加 `%AppData%\npm` 到系统 PATH 环境变量

**验证：**
```bash
which openclaw  # 应显示路径
```

**检查位置：** 安装后找不到命令 → 检查 PATH

---

#### 问题 3.2：配置文件损坏或不完整

**错误信息：**
```
config validation failed
```

**问题原因：**
- onboarding 向导未完成所有必填项
- 手动编辑导致语法错误

**解决方案：**

**验证配置文件：**
```bash
python3 -m json.tool ~/.openclaw/openclaw.json
```

**自动修复：**
```bash
openclaw doctor --fix
```

**手动修复：**
- 备份后删除 `~/.openclaw/openclaw.json`
- 重新运行 `openclaw doctor`

**检查位置：** 启动失败 → 检查配置文件语法

---

#### 问题 3.3：升级后配置不兼容

**错误信息：**
```
config validation failed: agent.* was moved, use agents.defaults
```

**问题原因：**
- 版本更新后配置结构变化
- 旧配置项被重命名或移除

**解决方案：**
```bash
# 自动修复配置
openclaw doctor --fix

# 验证配置
openclaw config get
```

**重要：** 每次升级后必须运行 `openclaw doctor --fix`

**检查位置：** 升级后启动失败 → 运行 doctor --fix

---

#### 问题 3.4：模型配置错误

**错误信息：**
```
model not allowed
model set failed: error: model not allowed
```

**问题原因：**
- 模型不在白名单中
- `agents.defaults.models` 未包含该模型

**解决方案：**

**检查白名单：**
```bash
openclaw config get agents.defaults.models
```

**添加模型到白名单：**
```bash
openclaw config set agents.defaults.models '["gpt-4", "claude-3-opus", "your-model"]'
```

**检查位置：** 模型调用失败 → 检查白名单

### 4️⃣ 端口冲突问题

#### 问题 4.1：默认端口被占用

**错误信息：**
```
Error: listen EADDRINUSE: address already in use :::18789
Port 18789 is already in use
```

**问题原因：**
- 其他进程占用 18789 端口
- 旧的 gateway 进程未清理

**解决方案：**

**查找占用进程：**

**macOS/Linux：**
```bash
lsof -ti:18789
# 或
ss -tlnp | grep 18789
```

**Windows：**
```powershell
netstat -ano | findstr :18789
```

**终止进程：**

**macOS/Linux：**
```bash
lsof -ti:18789 | xargs kill -9
```

**Windows：**
```powershell
taskkill /PID <PID> /F
```

**或更换端口：**
```bash
openclaw config set gateway.port 8080
```

**检查位置：** Gateway 启动失败 → 检查端口占用

---

#### 问题 4.2：多个 Gateway 实例冲突

**错误信息：**
```
gateway already running (pid xxx); lock timeout after 5000ms
another gateway instance is already listening
```

**问题原因：**
- 旧的 clawdbot-gateway 进程仍在运行
- 升级时未清理旧进程
- 多 profile 实例检测不区分

**解决方案：**

**查找所有 OpenClaw 进程：**
```bash
ps aux | grep openclaw
```

**清理旧进程：**
```bash
# 停止旧服务
systemctl --user stop clawdbot-gateway.service
systemctl --user disable clawdbot-gateway.service

# 删除 PID 锁文件
rm ~/.openclaw/gateway.pid

# 重启
openclaw gateway restart
```

**多 profile 修复（Issue #15970）：**
- 在 systemd service 的 ExecStart 中添加 `--profile` 参数

**检查位置：** 升级后启动失败 → 检查遗留进程

### 5️⃣ 依赖构建问题

#### 问题 5.1：sharp 构建失败

**错误信息：**
```
sharp: Please add node-gyp to your dependencies
error: /bin/sh: node-gyp: command not found
```

**问题原因：**
- 缺少预编译二进制文件
- 需要从源码编译但缺少工具

**解决方案：**

**macOS (Apple Silicon)：**
```bash
# 安装 Xcode 工具
xcode-select --install

# 设置环境变量
export SHARP_IGNORE_GLOBAL_LIBVIPS=1

# 重装 OpenClaw
npm install -g openclaw@latest
```

**如果仍失败：**
```bash
npm install -g node-gyp
npm install -g openclaw@latest
```

**检查位置：** npm install 失败 → 检查 sharp 构建错误

**错误信息：**
```
Error: Cannot find module 'some-package'
```

### 6️⃣ 升级问题

#### 问题 6.1：从 clawdbot 升级到 openclaw

**错误信息：**
```
gateway connect failed: Error: invalid connect params
data must NOT have additional properties
```

**问题原因：**
- 旧配置格式不兼容
- 旧 gateway 进程未停止

**解决方案：**

**升级步骤：**
1. 停止旧服务
   ```bash
   systemctl --user stop clawdbot-gateway.service
   ```

2. 升级
   ```bash
   npm install -g openclaw@latest
   ```

3. 运行迁移
   ```bash
   openclaw doctor --fix
   ```

4. 启动新服务
   ```bash
   openclaw gateway start
   ```

**检查位置：** 升级后启动失败 → 先停旧服务

### 7️⃣ 安全问题

#### 问题 7.1：CVE-2026-25253 RCE 漏洞

**严重程度：** CVSS 8.8（高危）

**影响版本：** 2026.1.29 之前的所有版本

**漏洞描述：**
- 跨站 WebSocket 劫持
- 可导致远程代码执行

**解决方案：**
```bash
# 立即更新到最新版本
npm update -g openclaw@latest

# 验证版本
openclaw --version  # 应为 2026.2.6 或更高
```

**检查位置：** 所有安装 → 确认版本 ≥ 2026.1.29

---

#### 问题 7.2：恶意 Skills

**问题描述：**
- ClawHub 上发现 341 个恶意 Skills
- 可能导致数据泄露

**解决方案：**
- 更新到 v2026.2.6+（集成 VirusTotal 扫描）
- 只安装官方 Skills
- 安装前检查源码和作者信誉

**检查位置：** 安装 Skills → 检查安全评分

## 📝 结论与建议

### 核心发现

1. **环境依赖是最大痛点**
   - Node.js 版本问题占比最高
   - 构建工具缺失导致安装失败

2. **权限问题容易被忽视**
   - 使用 sudo 安装会埋下隐患
   - 推荐使用 nvm 避免

3. **升级流程需要优化**
   - clawdbot → openclaw 升级常遇到遗留进程
   - 需要更清晰的升级文档

4. **安全问题严重**
   - CVE-2026-25253 需要立即修复
   - 恶意 Skills 风险需要关注

### 安装指南建议

**推荐安装流程：**
1. 检查 Node.js 版本（必须 ≥ 22）
2. 安装构建工具（build-essential / Xcode）
3. 使用 nvm 安装 Node（避免权限问题）
4. 安装 OpenClaw（`npm install -g openclaw@latest`）
5. 验证安装（`openclaw --version`，确保 ≥ 2026.2.6）
6. 运行诊断（`openclaw doctor --fix`）
7. 启动服务（`openclaw gateway start`）
8. 验证运行（`openclaw gateway status`）

**避免踩坑：**
- ❌ 不要使用 sudo 安装
- ❌ 不要使用 Node 18/20
- ❌ 不要跳过 `openclaw doctor --fix`
- ✅ 升级前先停止旧服务
- ✅ 每次升级后运行 doctor
- ✅ 只安装可信 Skills
