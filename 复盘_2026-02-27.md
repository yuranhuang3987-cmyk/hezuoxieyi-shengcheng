## 💡 可复用的知识沉淀

### 知识点 1：WSL2 代理配置标准流程

**场景**：WSL2 需要使用 Windows 代理上网

**步骤**：
```bash
# 1. 获取 Windows IP（动态）
WINDOWS_IP=$(cat /etc/resolv.conf | grep nameserver | awk '{print $2}')

# 2. 设置代理环境变量
export http_proxy="http://${WINDOWS_IP}:21882"
export https_proxy="http://${WINDOWS_IP}:21882"
export ALL_PROXY="socks5://${WINDOWS_IP}:21881"

# 3. 写入 ~/.bashrc 永久生效
# 4. 测试：curl https://api.ipify.org
```

**关键点**：
- Windows 代理必须允许局域网连接（0.0.0.0，不是 127.0.0.1）
- Windows 防火墙要开放端口
- IP 每次启动 WSL2 可能变化，所以要动态获取

**参考文档**：`D:\桌面\WSL2_VPN_配置完整SOP.md`

### 知识点 2：搜索工具选择策略

**决策树**：
```
需要搜索？
├─ 有 Brave API Key？
│  ├─ Yes → 使用 web_search（最快）
│  └─ No → 继续判断
├─ 需要精确页面内容？
│  ├─ Yes → 使用 web_fetch（最稳定）
│  └─ No → 使用 ddgs（可能超时）
└─ 搜索内容语言？
   ├─ 英文 → ddgs 成功率高
   └─ 中文 → ddgs 可能超时，优先 web_fetch
```

**工具对比**：
| 工具 | 速度 | 稳定性 | 中文支持 | API Key |
|------|------|--------|----------|---------|
| web_search | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ✅ | 需要 |
| web_fetch | ⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | ✅ | 不需要 |
| ddgs | ⭐⭐⭐ | ⭐⭐ | ⚠️ | 不需要 |

**最佳实践**：
1. 优先 web_fetch 抓取已知 URL
2. 搜索用英文关键词
3. ddgs 失败就换关键词重试
