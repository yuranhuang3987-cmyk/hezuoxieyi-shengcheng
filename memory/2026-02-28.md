### 协议生成器Web版 - 完整修复与测试（深度探索）

**问题发现：**
- 玉冉反馈 Web 版生成结果不正确
- 原因：Web 版代码与原版桌面程序逻辑不一致

**探索路径1：提取逻辑不一致**
- 问题：Web 版使用了复杂的多软件提取逻辑，导致提取结果错误
- 排查过程：
  1. 对比原版 main.py 和 Web 版 utils.py 的 extract_info()
  2. 原版逻辑：简单直接，从"软著名称"下一行直接取值
  3. Web 版逻辑：复杂循环，尝试提取多个软件，但过滤条件有误
- 解决：替换为原版简单直接的提取逻辑
- 结果：著作权人数量从 12 个修正为 10 个

**探索路径2：占位符识别失败**
- 问题：Web 版使用正则表达式替换，但正则模式不匹配模板中的实际占位符
- 排查过程：
  1. 用测试脚本提取模板中的实际占位符
  2. 发现模板使用中文引号：`"全方位鱼类去鳞资源整合与调配平台V1.0"`
  3. 原版代码预设的占位符：`"智能心理压力情绪疏导与调解平台V1.0"`（与模板不符！）
  4. 检查字符编码：
     - 左引号：U+201C（8220）
     - 右引号：U+201D（8221）
     - 正则字符集 `["""]` 无法正确匹配 Unicode 字符
- 解决方案：
  1. 动态从模板中提取实际占位符
  2. 使用正确的 Unicode 正则：`r'[\u201c\u201d"]([^"""]+V\d+\.\d+[^"""]*)[\u201c\u201d"]'`
  3. 同时提取日期、示例姓名、份数等占位符

**探索路径3：格式丢失问题（关键发现！）**
- 问题：输出文档的页面格式和段落内容被破坏
- 排查过程：
  1. 对比模板和输出文档的页面设置
     - 模板：8.27 x 11.69 英寸（A4纸）
     - 输出：8.50 x 11.00 英寸（Letter纸）
  2. 对比段落内容
     - 模板段落4：`经各方友好协商，就合作开发"全方位鱼类..."`
     - 输出段落4：`光伏建设对草地土壤微环境的影响监测平台V1.0`（只剩软件名！）
  3. 分析代码发现两个严重错误：
     - 错误1：创建新文档 `final_doc = Document()`
       - 问题：新文档使用默认页面设置，丢失模板格式
       - 解决：直接在模板文档上修改 `doc = Document(template_file)`
     - 错误2：替换函数 `runs[0].text = new_text`
       - 问题：整个段落被替换成 new_text，而不是只替换匹配部分
       - 解决：`runs[0].text = para.text.replace(old_text, new_text)`

**关键代码修复：**
```python
# 错误代码
runs[0].text = new_text  # ❌ 整个段落变成 new_text

# 正确代码
full_text = para.text.replace(old_text, new_text)  # ✅ 只替换匹配部分
runs[0].text = full_text
```

**技术要点总结：**
1. **Word文档处理原则**：
   - 直接修改模板而不是创建新文档，避免格式丢失
   - 页面设置、字体、样式等格式信息保存在文档级别
   
2. **文本替换原则**：
   - 必须保留完整段落文本，只替换匹配部分
   - 不能直接赋值 new_text，会破坏段落其他内容
   
3. **Unicode字符处理**：
   - Word文档中的引号可能是 Unicode 字符（U+201C/U+201D）
   - 正则表达式需要明确指定 Unicode 字符范围
   - 使用 `[\u201c\u201d"]` 而不是 `["""]`
   
4. **占位符提取策略**：
   - 动态从模板提取占位符比硬编码更可靠
   - 不同模板可能使用不同的示例内容
   - 原版代码中的占位符可能与实际模板不符
