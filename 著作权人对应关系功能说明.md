# 著作权人对应关系功能说明

## 🎯 功能更新

### 更新时间
2026-02-28 19:40

### 问题描述
之前的版本中，批量上传多个申请表时，所有软件的著作权人被合并成一个列表，无法区分每个软件对应的著作权人。

### 解决方案
现在每个软件都有自己独立的著作权人列表，系统会智能显示：

---

## 📋 显示逻辑

### 情况1：所有软件共享相同著作权人

**场景：** 同一个申请表中有多个软件，著作权人相同

**示例：** `光伏建设对草地土壤微环境的影响监测平台.docx`
- 软件1: 光伏建设对草地土壤微环境的影响监测平台V1.0
- 软件2: 草地光伏土壤微生态变化分析平台V1.0
- 软件3: 光伏建设草地土壤微环境监测软件V1.0
- **著作权人：** 10人（全部相同）

**显示效果：**
```
📋 所有软件共享同一组著作权人

1. 黑龙江八一农垦大学 (单位 - 显示名称)
2. 张裕杰 (个人 - 签名留空)
3. 陆硕 (个人 - 签名留空)
...
```

---

### 情况2：不同软件有不同的著作权人

**场景：** 批量上传多个申请表，每个申请表的著作权人不同

**示例：**
- 张三的申请表.docx（2个软件，著作权人：张三、李四）
- 王五的申请表.docx（1个软件，著作权人：王五、赵六）

**显示效果：**
```
⚠️ 不同软件对应不同的著作权人，请仔细核对

软件 1: 张三系统V1.0 [2 位著作权人]
  1. 张三 (个人 - 签名留空)
  2. 李四 (个人 - 签名留空)

软件 2: 李四平台V1.0 [2 位著作权人]
  1. 张三 (个人 - 签名留空)
  2. 李四 (个人 - 签名留空)

软件 3: 王五工具V1.0 [2 位著作权人]
  1. 王五 (个人 - 签名留空)
  2. 赵六 (个人 - 签名留空)
```

## 🔧 技术实现

### 后端修改

**文件：** `backend/app.py`

**修改内容：**
```python
# 之前：合并所有著作权人
all_owners = []
for owner in software_list[0]["owners"]:
    if owner not in all_owners:
        all_owners.append(owner)

# 现在：每个软件保留自己的著作权人
for sw in software_list:
    sw["software_owners"] = sw.get("owners", [])
```

**返回数据结构：**
```json
{
  "software_list": [
    {
      "software_name": "软件名称",
      "software_version": "V1.0",
      "owners": [
        {"name": "张三", "is_person": true},
        {"name": "李四单位", "is_person": false}
      ]
    }
  ]
}
```

### 前端修改

**文件：** `frontend/src/pages/Home.js`

**智能显示逻辑：**
```javascript
// 检查所有软件的著作权人是否相同
const allOwners = previewData.software_list.map(sw => sw.owners || []);
const allSame = allOwners.every(owners => 
  JSON.stringify(owners) === JSON.stringify(allOwners[0])
);

if (allSame) {
  // 相同：显示一次
  return <显示共享著作权人列表 />;
} else {
  // 不同：分别显示
  return <按软件分组显示著作权人 />;
}
```
